## Maestroとは

**Maestro**（マエストロ）は、複数のAIエージェント（AI assistants）を1つのチームとして管理・運用できるmacOS向けデスクトップアプリケーションです。

まるでオーケストラの指揮者のように、複数のAIを連携させて、複雑なタスクを効率的に処理できます。

### こんな方におすすめ

- AIを活用した作業を効率化したい方
- 複数のAIに異なる役割を持たせて協働させたい方
- プロジェクト管理とAI活用を組み合わせたい方
- チームでAIを活用したワークフローを構築したい方

---

## 主な機能

### 1. AIエージェントチームの編成

複数のAIエージェントを作成し、それぞれに役割を割り当てることができます。

| 役割 | 説明 |
|------|------|
| **BOSS（リーダー）** | チームを統括し、タスクの割り振りを行います |
| **Worker（作業者）** | 具体的なタスクを実行します |
| **Reviewer（レビュアー）** | 作業内容を確認・品質チェックします |
| **Specialist（専門家）** | 特定分野の専門的なタスクを担当します |

📸 ここにチーム編成画面のスクリーンショットを挿入

### 2. ミッションモード

AIエージェント間で自動的にタスクをリレーする「ミッションモード」を搭載。

- BOSSがタスクを分解してWorkerに指示
- Workerが完了したらReviewerが確認
- 承認後、BOSSに完了報告

人間の介入なしに、AIチームが自律的に作業を進めます。

📸 ここにミッションモードのワークフロー図を挿入

### 3. リアルタイムコミュニケーション

エージェント間のやり取りをリアルタイムで監視できます。

- 全エージェントの会話履歴を一覧表示
- どのエージェントが何を処理しているか可視化
- 必要に応じて人間が介入可能

📸 ここにコミュニケーション履歴画面のスクリーンショットを挿入

### 4. プロジェクト管理

複数のプロジェクトを作成・管理できます。

- プロジェクトごとにタスクを整理
- 進捗状況の追跡
- 作業履歴の保存

## システム要件

### 対応OS

| OS | バージョン | 対応状況 |
|----|----------|---------|
| macOS | 11.0 (Big Sur) 以降 | ✅ 対応 |
| Windows | - | ⏳ 今後対応予定 |
| Linux | - | ⏳ 今後対応予定 |

### 必要環境

- **ディスク容量**: 約100MB以上の空き容量
- **メモリ**: 4GB以上推奨
- **インターネット接続**: AI機能の利用に必要
- **Claude CLI**: Anthropic社のClaude CLIがインストールされていること

### 前提条件

Maestroを使用するには、**Claude CLI**がインストールされている必要があります。

Claude CLIのインストールと設定については、公式ドキュメントをご参照ください：

👉 [Claude CLI 公式ドキュメント](https://docs.anthropic.com/ja/docs/claude-cli)

> ⚠️ **注意**: Claude CLIのインストール、認証、APIキーの設定が完了してからMaestroをご利用ください。

1. **ダウンロード**

   このリポジトリから最新版をダウンロードしてください。

4. **初回起動**

   `Maestro.app` をダブルクリックして起動します。

## 初回起動時の設定

### ステップ1: セキュリティ警告の対処（初回のみ）

初めてMaestroを起動する際、macOSのセキュリティ機能により警告が表示される場合があります。

📸 ここにセキュリティ警告ダイアログのスクリーンショットを挿入

**対処方法:**

1. 「キャンセル」をクリック
2. 「システム設定」→「プライバシーとセキュリティ」を開く
3. 画面下部の「このまま開く」または「許可」をクリック
4. 再度 `Maestro.app` を起動

### ステップ2: 基本設定

初回起動後、以下の設定を確認してください。

📸 ここに設定画面のスクリーンショットを挿入

1. **言語設定**
   - 応答言語を選択（日本語/英語）

2. **通知設定**
   - タスク完了時の通知ON/OFF
   - BOSS応答受信時の通知ON/OFF

### ステップ3: AIエージェントの確認

初期状態で以下のエージェントが設定されています：

| エージェント名 | 役割 | 説明 |
|--------------|------|------|
| Maestro | BOSS | チームを統括するリーダーエージェント |
| Assistant | アシスタント | 自動応答システム用の補助エージェント |

必要に応じて、新しいエージェントを追加したり、既存のエージェントをカスタマイズできます。

📸 ここにエージェント一覧画面のスクリーンショットを挿入

## 基本的な使い方

### 1. プロジェクトを作成する

1. ホーム画面で「新規プロジェクト」ボタンをクリック
2. プロジェクト名と説明を入力
3. 「作成」をクリック

📸 ここにプロジェクト作成画面のスクリーンショットを挿入

### 2. タスクを開始する

1. プロジェクトを選択
2. 「新規タスク」をクリック
3. タスクの内容を入力
4. 「実行」をクリック

### 3. AIエージェントに指示を出す

1. 作業指示画面を開く
2. 指示を送るエージェントを選択（通常はBOSS）
3. テキストフィールドに指示内容を入力
4. `Ctrl+Enter` または「実行」ボタンで送信

> ✨ **ヒント**: 実行ボタンは入力フィールドの右下にあります。キーボードショートカット `Ctrl+Enter`（macOSでは `Cmd+Enter`）でも送信できます。

📸 ここに作業指示画面のスクリーンショットを挿入（実行ボタンの位置を矢印で示す）

**指示の例:**
```
プロジェクトのREADMEファイルを作成してください。
以下の内容を含めてください：
- プロジェクト概要
- インストール手順
- 使い方
```

### 4. 進捗を確認する

- **コミュニケーション履歴**: エージェント間のやり取りを確認
- **ログビューア**: 詳細な処理ログを確認
- **タスク一覧**: 各タスクの進捗状況を確認

📸 ここに進捗確認画面のスクリーンショットを挿入

### 5. AIエージェントをカスタマイズする

1. 「チーム編成」タブを開く
2. 編集したいエージェントを選択
3. 以下の項目を設定可能：
   - **名前**: エージェントの表示名
   - **アバター**: 絵文字アイコン
   - **役割**: BOSS / Worker / Reviewer / Specialist
   - **システムプロンプト**: エージェントの行動指針（下記参照）
   - **説明**: エージェントの担当業務

> **システムプロンプトとは？**
> AIの「性格」や「行動指針」を定義する文章です。例えば「丁寧な言葉遣いで回答する」「日本語で応答する」「コードを書くときは必ずコメントを入れる」などを設定できます。これにより、各エージェントに異なる個性や専門性を持たせることができます。

📸 ここにエージェント編集画面のスクリーンショットを挿入

## トラブルシューティング

### よくある問題と解決方法

#### Q1: アプリが起動しない

**症状**: `Maestro.app` をダブルクリックしても何も起こらない

**解決方法**:
1. macOSのバージョンが11.0以上か確認
2. システム設定 → プライバシーとセキュリティ で許可されているか確認
3. 右クリック →「開く」を選択して起動を試す

#### Q2: 「開発元を検証できません」と表示される

**症状**: セキュリティ警告でアプリが開けない

**解決方法**:
1. システム設定 → プライバシーとセキュリティ を開く
2. 「セキュリティ」セクションで「このまま開く」をクリック
3. 確認ダイアログで「開く」を選択

📸 ここにセキュリティ設定画面のスクリーンショットを挿入

#### Q3: AIエージェントが応答しない

**症状**: 指示を送っても応答がない

**解決方法**:
1. インターネット接続を確認
2. Claude CLIが正しくインストール・認証されているか確認
   - [Claude CLI 公式ドキュメント](https://docs.anthropic.com/ja/docs/claude-cli) を参照
3. アプリを再起動

#### Q4: エージェント間の通信が止まる

**症状**: ミッションモードでタスクが途中で止まる

**解決方法**:
1. ログビューアでエラーを確認
2. エージェントの設定（システムプロンプト）を確認
3. 手動で次のエージェントに指示を送信

#### Q5: 設定が保存されない

**症状**: 設定を変更しても次回起動時にリセットされる

**解決方法**:
1. 設定変更後、必ず「保存」ボタンをクリック
2. アプリケーションフォルダの書き込み権限を確認
3. `maestro` フォルダ内の設定ファイル（`app_settings.json`）が存在するか確認

#### Q6: 動作が重い・遅い

**症状**: アプリの動作が遅い、フリーズする

**解決方法**:
1. 不要なプロジェクトやタスクを整理
2. メモリ使用量を確認し、他のアプリを終了
3. アプリを再起動
4. macOSを再起動
